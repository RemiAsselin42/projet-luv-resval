## Prompt pour mettre à jour la documentation

Analyse les modifications qui ont été apportées aux fichiers récemment édités. Identifie les changements significatifs, les ajouts ou suppressions de code, et les ajustements de logique. Mets à jour la documentation du projet en conséquence, en veillant à ce que les nouvelles fonctionnalités, les changements de comportement, ou les ajustements de logique soient clairement expliqués dans la documentation. Assure-toi que la documentation est à jour, précise, et facile à comprendre pour les utilisateurs du projet.

## Prompt pour vérifier l'écart entre la documentation et le code

Analyse le code actuel du projet. Vérifie que la documentation du projet est en accord avec le code. Identifie tout écart ou incohérence entre la documentation et le code. Si des écarts sont identifiés, suggère des mises à jour spécifiques à apporter à la documentation pour qu'elle soit alignée avec le code actuel du projet ou vice versa. Assure-toi que la documentation reflète fidèlement le comportement, les fonctionnalités, et la logique du code actuel du projet.

## Prompt pour mettre à jour le AGENTS.md

Mets à jour le fichier AGENTS.md en fonction de l'état actuel du projet et de si des éléments de gestion du projet CRITIQUES sont à ajouter ou à mettre à jour. Assure-toi que les directives, les standards de code, les commandes par défaut, et les formats de réponse pour le travail de l'agent sont clairement définis et alignés avec les besoins actuels du projet. Attention le fichier AGENTS.md doit rester concis et orienté "guide de référence rapide" pour les contributeurs. Ne pas y inclure de longs paragraphes explicatifs qui peuvent être simplement remplacé par une analyse rapide du projet. Le AGENTS.md doit être un guide de référence rapide, pas une documentation détaillée du projet. Il doit faire moins de 200 lignes.
