## Prompt pour expliquer les modifications

Analyse et compare les modifications qui ont été apportées aux divers fichiers depuis le dernier commit. Identifie les changements significatifs, les ajouts ou suppressions de code, et les ajustements de logique. Fournis une synthèse claire des modifications effectuées et détails en une ligne les changements qui ont été apportés à chaque fichier modifié.

## Prompt pour évaluer les modifications

Analyse les modifications qui ont été apportées aux divers fichiers depuis le dernier commit. Identifie les changements significatifs, les ajouts ou suppressions de code, et les ajustements de logique. Que penses-tu de ces changements ? Sont-ils bénéfiques pour le projet ? Y a-t-il des risques ou des points d'attention à considérer suite à ces modifications ? Les fonctions ajoutées ou modifiées semblent-elles bien intégrées dans l'architecture globale du projet ? Y a-t-il des opportunités d'amélioration ou des suggestions que tu pourrais faire pour optimiser davantage le code ou la structure du projet à la lumière de ces changements ? Génère une liste de points positifs, de risques potentiels, et de suggestions d'amélioration basées sur les modifications apportées.

## Prompt pour appliquer les modifications

A partir des recommandations que tu viens de faire, applique les modifications nécessaires au code pour corriger les problèmes identifiés. Assure-toi que les changements sont bien intégrés dans l'architecture globale du projet et qu'ils respectent les standards de code établis. Après avoir appliqué les modifications, valide que le projet fonctionne correctement et que les changements ont eu l'effet désiré. Si des tests sont nécessaires pour valider les modifications, assure-toi de les exécuter et de vérifier que tous les tests passent avec succès.

## Prompt pour rédiger les commits

Analyse et compare les modifications qui ont été apportées aux divers fichiers depuis le dernier commit. Découpe les modifications en différents commits logiques (et exécute les commandes de commit), en regroupant les changements qui sont liés entre eux. Pour chaque commit logique, rédige un message de commit clair et descriptif en français, qui explique les changements apportés, les raisons de ces changements, et les impacts potentiels sur le projet. Assure-toi que les messages de commit sont concis mais informatifs, et qu'ils suivent les bonnes pratiques de rédaction de messages de commit. (cf : ./claude/skills/commit-work/README.md and SKILL.md)
